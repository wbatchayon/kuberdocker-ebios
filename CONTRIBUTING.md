# Guide de Contribution - Kuberdocker EBIOS

Merci de votre interet pour contribuer a ce projet EBIOS Risk Manager !

## Comment Contribuer

### Prerequisites
- Node.js 18+
- npm ou yarn
- Connaissances de base en React/TypeScript

### Etapes pour Contribuer

1. Forker le projet
2. Creer une branche (`git checkout -b feature/amelioration`)
3. Commiter vos changements (`git commit -m 'Ajout d'une feature'`)
4. Pusher sur la branche (`git push origin feature/amelioration`)
5. Creer une Pull Request

## Standards de Code

### Style de Code
- Utiliser ESLint pour le linting
- Suivre les conventions TypeScript
- Utiliser des hooks React pour la logique

### Tests
- Ajouter des tests pour chaque nouvelle fonctionnalite
- Maintenir la couverture de tests elevee
- Executer `npm run test:e2e` avant de pousser

### Documentation
- Documenter les nouvelles fonctionnalites
- Mettre a jour le README si necessaire
- Ajouter des commentaires dans le code complexe
